### 今回のスコープ: E4（関数合同公理）マッチング実装

### 背景

E4 は等号公理の一つで、関数の合同性を表す:

- 1引数関数 f: `∀x.∀y. x = y → f(x) = f(y)`
- 2引数関数 f: `∀x₁.∀x₂.∀y₁.∀y₂. x₁ = y₁ → (x₂ = y₂ → f(x₁,x₂) = f(y₁,y₂))`

現在 `identifyAxiom` は E1, E2, E3 のみ対応。E4 は「シグネチャ依存のため将来的に別途実装」とコメントされている。

### 影響するクエスト

- **peano-20**: `∀x.∀y. x = y → S(x) = S(y)` — E4(S) スキーマ
- **group-20**: `∀x.∀y. x = y → i(x) = i(y)` — E4(i) スキーマ
- **peano-11, 13, 14, 15**: E4(S) インスタンス `a = b → S(a) = S(b)` が含まれる（ただし E3 インスタンスも含むため E4 だけでは不十分）

### 実装計画

1. `inferenceRule.ts` に `matchE4` 関数を追加:
   - 論理式が `∀x₁...∀xₙ.∀y₁...∀yₙ. x₁=y₁ → (... → f(x₁,...,xₙ) = f(y₁,...,yₙ))` の形かチェック
   - また、インスタンス形 `t₁=s₁ → (... → f(t₁,...,tₙ) = f(s₁,...,sₙ))` もマッチ
2. `identifyAxiom` の equalityLogic セクションに E4 チェックを追加
3. `EqualityAxiomId` 型に "E4" を追加（既に含まれている — `"E1" | "E2" | "E3" | "E4" | "E5"`）

### テスト計画

- `inferenceRule.test.ts`: E4 マッチングテスト追加
  - E4(S): `∀x.∀y. x = y → S(x) = S(y)` がマッチすること
  - E4(i): `∀x.∀y. x = y → i(x) = i(y)` がマッチすること
  - E4(S) インスタンス: `S(0) + 0 = S(0) → S(S(0) + 0) = S(S(0))` がマッチすること

  - 非 E4 の式がマッチしないこと
