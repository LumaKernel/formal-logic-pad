# ProofWorkspace メニュー体験の設計

## 背景・課題

Auto Layout が「謎のチェックボックス」として存在しており、直感的でない。
これを機に、ワークスペース全体のメニュー・アクション体験を整理する。

## 設計方針

### 原則

1. **アクションは文脈に応じた起点から起動する** — グローバルに常時表示するのは最小限に
2. **セレクションベースの操作を基本にする** — 「選んでから操作」のパラダイム
3. **同じアクションに複数の起点を提供** — コンテキストメニュー、キーボードショートカット、コマンドパレットの3つを基本に
4. **ツールバーは最も頻繁に使う推論規則の適用のみ** — レイアウト操作はツールバーに置かない

### Auto Layout の改善案

**問題:** チェックボックスで「モード」を切り替えるのは不自然。ユーザーは「今すぐレイアウトして」と思うことが多い。

**方針:**

- チェックボックスを**削除する**
- 代わりに以下の起点を提供:
  1. **コマンドパレット** (`/` → "Auto Layout" / "整列")
  2. **キャンバスコンテキストメニュー** ("整列" サブメニュー)
  3. **キーボードショートカット** (例: Cmd+Shift+L)
- 「整列」アクションは**ワンショット**（現在のノード配置にレイアウトを一度適用）
- 方向の選択は整列ダイアログ/サブメニューで行うか、デフォルトを使う

#### 整列の種類

| アクション        | 対象             | 説明                                     |
| ----------------- | ---------------- | ---------------------------------------- |
| ツリー整列(上→下) | 選択 or 全ノード | 選択ノードがあれば選択のみ、なければ全体 |
| ツリー整列(下→上) | 選択 or 全ノード | 同上                                     |

#### 起点の統合

| 起点                           | 操作                                 |
| ------------------------------ | ------------------------------------ |
| コマンドパレット               | "ツリー整列" → 方向選択 → 実行       |
| キャンバスコンテキストメニュー | "ツリー整列" → 方向サブメニュー      |
| キーボードショートカット       | Cmd+Shift+L → デフォルト方向で即実行 |

### セレクションと整列の関係

**「整列はセレクションがあってこそ」の考え:**

- セレクションがある場合 → 選択ノードのみを対象にレイアウト適用
- セレクションがない場合 → 全ノードを対象にレイアウト適用
- これにより「部分整列」も自然に実現

### メニューの宣言的管理

メニュー項目を宣言的に定義し、各起点（コンテキストメニュー、コマンドパレット、ショートカット）から参照するアーキテクチャ:

```
MenuAction定義（宣言的データ）
  ├── id: "tree-layout-top-to-bottom"
  ├── label: { en: "Tree Layout (Top→Bottom)", ja: "ツリー整列（上→下）" }
  ├── shortcut: "mod+shift+l"
  ├── icon?: ...
  ├── contexts: ["canvas-menu", "command-palette"]
  ├── enabled: (state) => boolean
  └── execute: (state) => newState
```

これを以下の場所から参照:

- コンテキストメニューレンダラー: `contexts` に "canvas-menu" を含むものをフィルタ
- コマンドパレット: `contexts` に "command-palette" を含むものをフィルタ
- キーボードショートカット: `shortcut` が定義されているものを登録

**段階的に移行**: まず整列アクションから始めて、既存のアクションは後から順次移行。

## 実装ステップ（この設計の実現）

1. **Auto Layout チェックボックスを削除**
2. **整列アクションをコマンドパレットに追加**
3. **整列アクションをキャンバスコンテキストメニューに追加**
4. **キーボードショートカットを追加** (Cmd+Shift+L)
5. **セレクション対応**: 選択ノードがあれば部分整列、なければ全体整列
6. **(将来)** 宣言的メニューアクション管理の共通基盤を構築
