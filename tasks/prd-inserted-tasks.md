# 差し込みタスク

- [ ] 設計がミスっていそう。MP適用があるものについて、そのコネクションを消したら、もう片方の適用される側などもコネクションが消えて、ノードはderivedではなく、浮いてるノードになるべきだ。
    公理としてマークできる機能はいらない気がしてきた。
- [ ] ゴールも、ゴールノードではなく、サイドパネルや一覧が提示されていて、それをキャンバスのどこかで、（使っていい公理だけで）証明されればゴールということにして、公理だとかゴールだとか言う内部ステートもやめよう。
- [ ] 同一のノードをマージできるようにしよう。このとき、上からのコネクション（derive）を持つもの同士はどちらから引っ張ればいいか自明でないので、先に選択された方など、リーダー的なのを決めて、そちらからderiveされたことにしよう。逆に、定理として利用されてる立場のコネクションはどちらも保持され、正しくマージされるべきだろう。
